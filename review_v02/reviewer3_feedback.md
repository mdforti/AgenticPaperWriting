# Referee Report — Reviewer 3 (Statistical ML Methodology)

**Manuscript:** "Data-efficient machine-learning of complex Fe–Mo intermetallics using domain knowledge of chemistry and crystallography"
**Journal:** Physical Review B
**Reviewer scope:** statistical learning methodology, validation protocol, significance testing, uncertainty quantification, reproducibility.

## (i) Summary

The authors train a VotingRegressor ensemble (KRR-RBF + RF-200 + MLP-64) on 209 simple-TCP Fe–Mo structures, evaluate on a 53-structure held-out test set, and transfer to a 70-structure validation set of complex TCP phases (R, M, P, δ). They compare three descriptor families — BOP moments, ACE, SOAP — aggregated via a coordination-number-resolved averaging (CNAV) scheme, and use the ML-predicted energies inside a Bragg–Williams compound-energy formalism to recover R-phase Wyckoff occupancies. The headline statistical claims are: (a) BOP > ACE > SOAP, with "p < 0.01, bootstrap test"; (b) CNAV improves RMSE by 20–40% over per-structure averaging; (c) the BOP advantage "persists at all training-set sizes"; (d) MAD of 0.04 in Mo site fraction vs. XRD.

The pipeline is sensible and the engineering choices (stratified split, RBF-KRR, 200-tree RF, voting ensemble, EOS-fitted energies) are reasonable for this data regime. However, the statistical reporting around the central comparisons is too thin for PRB. Several quantitative claims — most importantly "p < 0.01" for BOP > ACE, the "20–40%" CNAV gain, and "agreement is quantitative" for occupancies — are presented without the test specification, effect size, confidence interval, or multiple-comparison correction needed for a reader to assess them. Below I list the items that, in my view, must be addressed before this is publishable as a methodological contribution.

## (ii) Major comments

1. **Significance test for "BOP > ACE > SOAP" is under-specified (Results §test_performance, line 65).** The manuscript states only "The BOP advantage over ACE is statistically significant (p<0.01, bootstrap test)". What is the test? Paired or unpaired? On what statistic — per-structure squared errors, RMSE differences across resamples, or CV-fold RMSEs? How many bootstrap resamples (B)? Was BOP–SOAP also tested? Were the three pairwise comparisons (BOP–ACE, BOP–SOAP, ACE–SOAP) corrected for multiple testing (Holm or Benjamini–Hochberg)? With N_test = 53 and three pairwise tests, an uncorrected p-value is not interpretable. **Remedy:** report a paired bootstrap on per-structure absolute errors (B ≥ 10 000), give the mean RMSE difference with a 95% percentile-bootstrap CI and effect size (e.g. Cohen's d on residual magnitudes or the bootstrap probability P(RMSE_BOP < RMSE_ACE)), and apply Holm–Bonferroni across the three comparisons. Alternatively, the corrected resampled t-test of Nadeau & Bengio (2003) is the standard for CV-fold comparisons in small-data regimes and is more appropriate than a naive paired t.

2. **The ±3, ±4, ±5 meV/at error bars in Table 1 are unattributed.** Are these CV-fold standard deviations, repeated-split standard deviations, bootstrap CIs, or something else? Reporting "28 ± 3" without specifying the underlying distribution is non-actionable. **Remedy:** state explicitly (e.g. "RMSE ± 95% percentile bootstrap CI over B = 10 000 resamples of the 53-structure test set"). Add 95% CIs for R² as well; R² on N = 53 has substantial sampling noise.

3. **Feature selection nesting is asserted but not demonstrated.** Methods §Feature Selection states "Forward recursive feature selection … nested inside the 5-fold CV loop: for each fold, selection was applied to the training portion only." Yet the reported "selected dimension (~80, ~200, ~100)" is a single number per descriptor in Table 1, which suggests selection on the full training set was then refit. If selection was truly per-fold, the selected feature set varies between folds, and the reported "~80" should be a distribution. **Remedy:** report median and IQR of selected dimension across folds; show overlap/Jaccard index of feature sets across folds (stability analysis); and confirm in one sentence that the test set was never seen by the selector. Forward selection is famously unstable in p ≫ n regimes (400–1800 features, 209 samples) — without stability-selection or repeated CV, the apparent "80 features" is essentially noise.

4. **Hyperparameter tuning and feature selection use the same 5-fold CV — double dipping.** The Methods section uses the same "5-fold CV" loop for (i) RBF-KRR α and γ grid search, (ii) RF depth/leaf grid search, (iii) MLP α grid search, and (iv) forward feature selection. If these share folds, the CV RMSE is an optimistic estimate of generalisation, and the test-set RMSE in Table 1 is the only honest number — yet the headline error bars and "p < 0.01" use the (potentially leaked) CV machinery. **Remedy:** describe the protocol as proper nested CV — outer 5-fold for evaluation, inner CV for tuning and selection — or, if the held-out 53-structure test is the only evaluation, drop the ± and report a single test RMSE plus a paired bootstrap CI on the test residuals.

5. **The simple-→-complex transfer is presented as the central result, but transfer-set RMSEs in Table 2 are reported without any uncertainty.** Numbers like "R: 35, M: 40, P: 38, δ: 42" (BOP) lack CIs, fold counts, or seed-variance. For a transfer claim, the reader needs to know whether the BOP–ACE gap (35 vs. 45 on R, 40 vs. 50 on M, …) exceeds the variability across retrainings. **Remedy:** repeat training with ≥ 20 random seeds and report mean ± 95% CI per phase per descriptor, then a paired test (e.g. Wilcoxon signed-rank across the 70 validation residuals) per pair. State the test name and the corrected p-values.

6. **Group/structural leakage between simple-TCP training and complex-TCP validation is not analysed.** Complex phases (R, M, P, δ) are built from the same Frank–Kasper polyhedra (CN 12, 14, 15, 16) as the simple phases, and CNAV encodes precisely those CN groups. This is the *point* of the method, but it also means that the "independent" validation set may share local environments with training. **Remedy:** quantify per-atom descriptor overlap (e.g. nearest-neighbour distance in CNAV feature space from each validation atom to the training set), and show how RMSE varies as a function of this distance. This is the standard out-of-domain check (Janet et al., Sutton et al.) and would directly support — or qualify — the transferability claim.

7. **No predictive uncertainty is propagated to the thermodynamic analysis.** Figure 5 shows "error bars indicate the uncertainty from ML prediction" for R-phase occupancies, but the source of those error bars is not described, and the MAD = 0.04 quoted in Table 3 has no CI. With residual variances of order (35–42 meV/at)², the propagated uncertainty on a site fraction obtained by minimising Eq. (1) is non-trivial. **Remedy:** sample formation energies from a bootstrap or ensemble-disagreement distribution (e.g. 200 retrainings, or per-model VotingRegressor spread), re-solve Eq. (1) for each draw, and report 95% intervals on each site fraction. Conformal prediction over the test residuals is an even cleaner option and is now standard.

8. **Trivial baselines are missing.** The only weak baseline is "Dataset (magpie), 55 ± 6 meV/at" in Table 1. A reader cannot tell what fraction of the BOP RMSE reduction is due to (a) the descriptor, (b) CNAV, (c) the ensemble, or (d) feature selection. **Remedy:** report (i) mean-of-training predictor, (ii) composition-only linear/ridge model, (iii) single-descriptor KRR with no CNAV and no feature selection, (iv) BOP+CNAV with each individual model (KRR, RF, MLP) separately, (v) BOP+CNAV ensemble with no feature selection. A small ablation table separating these effects is essential to support the claim that "domain knowledge is essential".

9. **"CNAV improves RMSE by 20–40%" (Methods §CNAV) is reported as a range without a test or a paired comparison.** What is the unit of comparison — RMSE on the test set, CV-mean RMSE, validation RMSE? **Remedy:** show a paired table (per descriptor: no-CNAV vs. CNAV vs. per-site) with bootstrap CIs and a paired test on per-structure errors. The 10% gain over per-site descriptors quoted in Methods is particularly fragile and needs a CI.

10. **Learning curves (Fig. 6) need clarification.** The caption says "±1 standard deviation over 10 repeated splits". Ten repeats is the minimum acceptable; 20–30 is preferable. More importantly, the *claim* that "the BOP advantage persists at all training-set sizes" requires a paired test at each N, not visual inspection of overlapping bands. **Remedy:** at each N ∈ {50, 100, 150, 209}, perform a paired bootstrap of test residuals and report the p-value (with Holm correction across the four sizes) plus effect size.

## (iii) Minor comments

1. Methods §Model Training states random seeds "42" for numpy, sklearn and Python. A single seed across the entire pipeline does not protect against seed-sensitivity; please confirm that the reported RMSEs are stable when the seed is varied (e.g. seeds 0–9, report median and IQR).
2. The scikit-learn version (1.2), Python (3.10), and conda env (`datasets_ml`) are stated in CLAUDE.md but the manuscript itself only cites "scikit-learn (v1.2)". Please add Python version, OS, and BOPfox, DScribe (v2.0 is given), python-ace versions explicitly in Methods.
3. Methods §Ensemble: "5–15% lower than the best individual model" — give a CI and clarify whether this is across descriptors or within a single descriptor.
4. Table 1 column "Dim. (selected)" uses tildes (~80, ~200, ~100). For reproducibility, give exact counts (or per-fold distribution as in major #3).
5. Methods §Thermodynamics: the SLSQP global optimum from 10 random starts is reassuring but not a proof of global optimality for a non-convex objective. Please confirm via a coarse grid scan or stochastic search.
6. The bibliography contains `\cite{dsribe}` (typo for DScribe) — check.
7. Parity plots (Fig. 3) and residual histograms per phase would help readers visualise the heteroscedasticity hinted at by per-phase RMSE differences.
8. The Joubert 1993 XRD reference (Table 3) should include the experimental uncertainty on each site fraction so the |Δ| column can be interpreted relative to measurement noise.
9. The data/code DOI (10.5281/zenodo.19427673) should be checked at proof; please also state whether the deposit includes the exact seeds, the per-fold selected feature sets, and the fitted model pickles.
10. "Test set (53 structures) and independent validation set (70 structures)" — state in Methods whether the 70 validation structures were generated, descriptor-computed, and split *before* any model development, to rule out implicit selection-on-the-test-set.

## (iv) Recommendation

**Major revision.** The methodological framework is sound and the result is potentially valuable to the PRB readership, but the central statistical claims — the BOP > ACE > SOAP ordering, the CNAV improvement, the simple-→-complex transfer, and the occupancy agreement — are presently stated without effect sizes, confidence intervals, correctly specified tests, or proper multiple-comparison control. These are fixable with re-runs of the existing pipeline and additional bootstrap/ablation tables; no new DFT is required.
